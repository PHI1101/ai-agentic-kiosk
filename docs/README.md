# AI Kiosk 프로젝트 분석

## 1. 개요 (Overview)

이 프로젝트는 'AI Kiosk'로, 단순한 챗봇을 넘어 사용자의 주문 과정을 처음부터 끝까지 주도하고 관리하는 **AI 에이전트** 시스템입니다. 사용자가 앱에 진입하는 순간부터 AI 에이전트를 소개해 역할을 명확히 하고, 음성 인식과 대화형 인터페이스를 통해 주문을 처리합니다.

핵심 목표는 일반 사용자는 물론, 시각/청각 장애인 등 정보 접근에 어려움을 겪는 사용자들도 편리하게 주문할 수 있는 '배리어프리(Barrier-Free)' 키오스크를 구현하는 것입니다.

## 2. 주요 기능 (Core Features)

- **AI 에이전트 소개 (시작 화면)**
  - 앱 실행 시, AI 에이전트의 역할과 기능을 소개하는 랜딩 페이지를 통해 사용자에게 친근한 첫인상을 제공합니다.
  - 사용자는 '주문 시작하기' 버튼을 통해 능동적으로 주문 프로세스를 시작합니다. (`HomePage.tsx`)

- **실시간 AI 상태 시각화 (아바타)**
  - 주문 화면에 AI 에이전트 아바타를 배치하여, 현재 AI의 상태(대기, 듣는 중, 생각 중)를 시각적으로 보여줍니다.
  - 이를 통해 사용자는 AI가 자신의 말을 듣고 있는지, 처리 중인지 직관적으로 인지할 수 있습니다. (`AiAgentAvatar.tsx`)

- **대화형 주문 플로우**
  - **음성 인식**: `react-speech-recognition`을 통해 사용자의 음성을 텍스트로 변환합니다.
  - **주문 확인**: AI가 사용자의 메뉴 요청을 인식하면, "장바구니에 담을까요?"라고 되물어 주문 실수를 방지합니다.
  - **상태 관리**: 사용자가 "네"라고 답하면 `orderStore`의 상태를 업데이트하여 장바구니에 상품을 추가합니다.

- **시각적 주문 현황 (장바구니)**
  - 사용자가 주문한 메뉴, 수량, 가격이 담긴 장바구니(주문 현황)를 화면에 실시간으로 표시합니다.
  - 총 주문 금액을 즉시 확인할 수 있으며, '주문 확정' 및 '전체 취소'가 가능합니다. (`OrderSummary.tsx`, `orderStore.ts`)

- **백엔드 연동**
  - Django 백엔드가 스타벅스, 메가커피 등 여러 프랜차이즈의 메뉴 데이터를 관리하며, 프론트엔드에서 요청한 주문 명령을 처리합니다.

## 3. 서비스 대상 (Target Audience)

이 서비스는 키오스크를 사용하는 모든 고객을 대상으로 하지만, 특히 다음과 같은 사용자들에게 특화된 가치를 제공하는 것을 목표로 합니다.

- **일반 사용자**: 음성을 통해 빠르고 편리하게 주문하고자 하는 모든 고객.
- **장애인 사용자**:
  - **시각 장애인**: 화면을 보지 않고도 음성 안내와 음성 명령만으로 메뉴 선택부터 주문까지 완료할 수 있도록 지원합니다.
  - **청각 장애인**: 명확하고 직관적인 채팅 인터페이스를 통해 소리 없이도 원활하게 주문할 수 있도록 지원합니다.
  - **신체 장애인**: 휠체어 사용자 등을 고려한 UI/UX 설계를 통해 물리적 접근성을 높입니다.

## 4. 기술 스택 (Tech Stack)

- **Frontend**:
  - React, TypeScript
  - **React Router DOM**: 시작 화면과 주문 화면 간의 페이지 라우팅
  - **Zustand**: 채팅(`chatStore`) 및 장바구니(`orderStore`) 상태 관리
  - **Material-UI**: UI 컴포넌트 및 디자인 시스템
  - **React Speech Recognition**: 음성 인식
  - **Axios**: 백엔드 API 통신

- **Backend**:
  - Django, Python
  - Django REST framework (API 구축)

- **Deployment**:
  - Vercel (Frontend), Railway (Backend) - `vercel.json` 설정으로 유추
